# 技术方案

## 推荐方案

第一版建议使用静态网站方案，内容用 Markdown 管理，不购买传统服务器。

如果希望国内外都尽量稳定访问，推荐路线是：

1. 本地先用 Astro 搭建个人网站。
2. 内容确认后购买域名。
3. 正式上线时优先选择中国内地静态托管 + CDN。
4. 如果暂时不想备案，可以先部署到 Cloudflare Pages / Vercel / Netlify，但国内访问稳定性不作为强保证。

## 推荐技术栈

- 框架：Astro。
- 内容：Markdown/MDX。
- 样式：原生 CSS，后续需要时再引入 Tailwind CSS。
- 部署：阿里云 OSS 静态网站托管 + CDN，或 Cloudflare Pages / Vercel 作为轻量备选。

## 为什么这样选

- 个人展示网站以内容为主，不需要复杂后端。
- Markdown 方便长期维护项目资料。
- 静态部署成本低、速度快、稳定。
- 后续如果需要博客、双语或更多页面，也容易扩展。
- 不需要维护 Linux 服务器、安全补丁、数据库和后台系统。

## 初始页面结构

- `/`：首页。
- `/projects`：项目列表。
- `/projects/[slug]`：项目详情。
- `/about`：关于我。

## 内容数据结构

每个项目建议包含：

- 项目名称。
- 时间。
- 角色。
- 简短摘要。
- 背景与目标。
- 主要工作。
- 成果与链接。
- 展示图片。

## 不建议第一版购买

- 云服务器 / VPS。
- WordPress 主机。
- 数据库。
- 后台管理系统。
- 复杂的商业建站套餐。
