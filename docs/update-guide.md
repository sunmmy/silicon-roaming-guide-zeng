# 网站更新指南

这份文档给未来的自己看：以后要更新“硅基漫游指南曾”时，尽量按固定流程来，避免每次重新摸索。

## 最理想的更新流程

```text
写内容 -> 放图片 -> 本地预览 -> 同步到 GitHub -> 网站自动更新
```

## 当前线上地址

```text
https://silicon-roaming-guide-zeng.zengli100.workers.dev
```

这是 Cloudflare 提供的临时访问地址。后续绑定正式域名后，再把正式域名补充到这里。

## 第一次上线前需要做

1. 准备一个 GitHub 账号。
2. 在 GitHub 上创建一个新仓库。
3. 把本地项目同步到这个仓库。
4. 选择 Cloudflare Pages 或 Vercel，连接这个 GitHub 仓库。
5. 平台会自动构建并发布网站。

当前这一步已经完成，后续重点是内容更新和正式域名。

## 以后更新项目

1. 在 `content/projects/` 新增或修改项目内容。
2. 把项目截图、封面、公众号二维码等图片放到固定图片目录。
3. 本地预览网站。
4. 确认没问题后同步到 GitHub。
5. 等待部署平台自动发布。

## 以后新增一个 lab 项目

建议每个项目都整理成一篇项目说明，至少包含：

- 项目名称。
- 项目是什么。
- 为什么做。
- 我做了什么。
- 最有趣的地方。
- 可以展示的截图、链接或报告。

## 不要做的事

- 不要直接手动上传 `dist/` 文件夹。
- 不要把 `node_modules/` 上传到仓库。
- 不要为了个人网站购买服务器。
- 不要每次更新都重新配置部署平台。

## 当前推荐平台

第一选择：

- GitHub + Cloudflare Pages。

备选：

- GitHub + Vercel。

两个方案都可以做到：本地更新、同步仓库、网站自动发布。
