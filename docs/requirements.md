# 需求文档

## 文档状态

- 版本：v0.2 草案
- 当前阶段：需求确认中
- 原则：先明确网站要解决什么问题，再决定页面、功能和技术实现。

## 一句话目标

建设一个稳定、低维护成本的个人实验室网站，用来展示个人用 AI 做出的探索性项目、创作实验和有趣想法。

网站不优先服务求职简历，而是作为一个“个人 lab 展示空间”：让别人看到一个想象力丰富、有执行力、有趣，并且对 AI、创造和创作有长期经验的人。

## 网站名称

初步确定：

- 名称：硅基漫游指南曾
- 关联身份：与同名公众号保持一致
- 命名含义：
  - “硅基”呼应 AI、机器、生成式工具和技术实验。
  - “漫游”表达探索、游走、好奇和个人实验。
  - “指南曾”包含作者姓氏“曾”，同时与“指南针”形成谐音。

这个名字适合当前定位：它不像传统简历站，也不像纯技术博客，而更像一个围绕 AI、创作、实验和个人探索展开的长期栏目。

## 背景

目前个人项目、实验和创作可能分散在本地 `lab` 文件夹、临时页面、报告或代码目录中，不方便长期整理和对外展示。这个网站应成为一个轻量的个人实验室入口：把未来 `lab` 里的项目逐步整理出来，形成一个可被访问、可被理解、可持续更新的作品档案。

## 目标用户

第一优先级：

- 对个人 AI 实验、创意项目、生成式项目感兴趣的人。
- 偶然看到某个项目后，想进一步了解作者和更多实验的人。
- 可能的合作者、朋友、同行或对 AI 创作有兴趣的人。

第二优先级：

- 对具体项目、模拟、工具、信息图、游戏或创作过程感兴趣的人。
- 未来的自己，用来回顾个人积累。

暂不优先考虑：

- 明确的招聘筛选场景。
- 传统学术论文展示场景。
- 需要登录、互动、评论或后台管理的用户。

备注：当前不预设一个非常明确的受众。网站更像开放的个人实验室，允许“感兴趣的人自然找到感兴趣的东西”。

## 网站要回答的问题

访客进入网站后，应能快速回答：

- 这个人是谁？
- 这个人为什么有趣？
- 他如何使用 AI 做项目、实验和创作？
- 做过哪些代表性个人项目？
- 每个项目中本人承担了什么工作？
- 有哪些可以查看的成果、链接或材料？
- 如果我对某个项目感兴趣，如何继续查看或联系？

## 第一版范围

第一版只做个人展示网站的核心能力。

必须有：

- 首页：展示姓名/昵称、身份方向、一句话介绍、精选项目入口。
- 关于/联系页：更完整的个人介绍、创作方向、联系方式和公众号二维码。
- 项目列表页：展示所有代表性项目。
- 项目详情内容结构：每个项目至少包含背景、目标、我的工作、成果、链接。
- 响应式布局：电脑和手机都能正常阅读。
- 低维护内容结构：后续新增项目不需要大改页面。

建议有：

- 精选项目模块。
- 项目标签或类别。
- 项目标签或类别，例如 AI 实验、模拟、可视化、小游戏、写作生成、工具。
- 社交链接，例如 GitHub、邮箱、公众号、知乎等。

暂不做：

- 登录注册。
- 后台管理系统。
- 评论区。
- 数据库。
- 复杂动画。
- 在线支付或商业转化功能。
- 过多栏目。
- 第一版不放简历下载。
- 第一版不放写作、论文、研究笔记的独立栏目。

## 内容栏目

### 首页

首页重点是“快速建立第一印象”：这是一个会冒火花的个人 AI 实验室，而不是传统简历页。

建议包含：

- 姓名或公开昵称。
- 当前身份或方向，例如“AI 创作与个人实验室”。
- 一句话介绍，要有动感、实验感和一点爆炸感，避免平淡说明文。
- 3-5 个关键词，例如 AI、创作、模拟、想象力、个人项目。
- 1-3 个精选项目。
- 联系入口。

首页文案气质参考：

- 科学怪人实验室。
- 生活大爆炸式的理工脑洞和日常幽默。
- Rick and Morty 式的漫游、荒诞、实验和失控感。
- 但表达上仍要可读、可信，不做低幼化或纯搞怪。

一句话介绍候选方向：

- 在 AI 的电火花里，把奇怪想法接上线、跑起来、再看看它会撞出什么。
- 一个把脑洞接进 AI 反应堆的个人实验室。
- 这里存放一些被 AI 点燃的想法、模拟、怪实验和意外产物。
- 把想象力扔进硅基反应釜，看看会生成什么新物种。
- 一间不太安分的硅基实验室，收容被 AI 点燃的怪点子和意外产物。

当前推荐：

- 一间不太安分的硅基实验室，收容被 AI 点燃的怪点子和意外产物。

### 关于/联系

关于/联系页重点是“让别人更完整地理解你，并知道如何找到你”。不单独拆成多个页面。

建议包含：

- 个人背景。
- 与 AI、创作、项目实验相关的个人背景。
- 当前关注的问题：AI 如何帮助个人快速把想法做成可运行的项目。
- 不必写成正式履历，可以更像个人实验室说明。
- 联系方式。
- 公众号二维码。

### 项目

项目页重点是“讲清楚每个实验是什么、为什么有趣、你是怎么把它做出来的”。

每个项目建议包含：

- 项目名称。
- 时间。
- 项目类型。
- 我的角色。
- 简短摘要。
- 背景与目标。
- 我的具体工作。
- 成果或影响。
- 图片、链接、代码、文档或演示。

第一版项目：

- `2605-和平博弈`：和平博弈 / 自动化 LLM 信任摧毁模拟器。
  - 类型：AI agent 模拟、政治叙事/信任机制实验、个人研究型项目。
  - 当前已有材料：README、静态页面、实验输出、报告、可视化素材。
  - 展示重点：用 LLM 驱动多个立场智能体，观察同一事件如何被不同位置解释，以及信任如何在解释链条中被消耗。

### 写作/研究

第一版暂不加入独立写作/研究栏目。

适合展示：

- 论文。
- 文章。
- 研究笔记。
- 案例分析。
- 公开演讲或报告。

## 内容优先级

第一优先级：

- 个人介绍。
- 代表性项目。
- 联系方式。

第二优先级：

- 简历。
- 写作/研究。
- 项目截图和封面。

第三优先级：

- 更细的分类系统。
- 多语言。
- 复杂视觉效果。

## 风格需求

已确认方向：

- 简洁、清楚、可信，但可以有一点好玩和个人气质。
- 偏“个人实验室 / AI 创作档案”，不做传统求职简历网站，也不做强营销感页面。
- 阅读体验优先，视觉设计服务内容。
- 国内外访问都应尽量稳定。
- 中文优先。
- 让人感受到：想象力丰富、有执行力、有趣、熟悉 AI 创作。
- 可以和同名公众号形成呼应，但网站不应只是公众号文章索引，而应承担“项目陈列室”和“个人实验室入口”的角色。
- 公众号入口放在“联系我”区域，第一版用二维码呈现即可，不单独做公众号栏目。

待确认：

- 联系方式公开哪些。
- 视觉气质更偏“实验室档案”“有趣作品集”还是“克制技术感”。

## 技术与维护需求

基本要求：

- 不购买传统服务器作为第一选择。
- 使用静态网站方案，降低维护成本。
- 页面加载快。
- 方便后续新增项目和修改文字。
- 可以绑定自己的域名。
- 必须支持方便同步：用户在电脑上改完内容后，应能通过固定流程快速发布到网站。
- 不应依赖手动上传服务器文件。
- 项目内容应尽量用 Markdown 管理，减少直接改代码。

长期要求：

- 支持迁移到更稳定的国内托管方案。
- 支持后续扩展博客、英文版或更多项目页。
- 支持 GitHub + 自动部署平台的更新模式。

## 非功能需求

- 稳定：不依赖复杂后端服务。
- 低成本：个人网站早期尽量使用免费或低价方案。
- 易维护：内容更新尽量简单。
- 易同步：从本地电脑更新内容到线上网站的步骤要固定、短、可重复。
- 可访问：手机和电脑都能正常阅读。
- 可扩展：后续可以逐步加文章、简历、双语和更多项目。

## 阶段规划

### 阶段 1：需求确认

- 明确网站定位。
- 明确第一版功能范围。
- 明确第一批内容。
- 明确是否需要联系方式和公开边界。

### 阶段 2：内容整理

- 完成个人介绍。
- 先整理 `2605-和平博弈`，作为第一版样板项目。
- 后续再补充 2-4 个新的 lab 项目。
- 收集图片、截图、链接。

### 阶段 3：第一版网站

- 搭建首页、关于页、项目页。
- 接入真实内容。
- 完成手机和电脑预览。
- 把项目内容改成 Markdown 驱动，方便后续维护。

### 阶段 4：上线

- 选择域名。
- 选择部署平台。
- 完成发布。
- 后续根据访问稳定性决定是否备案和迁移。

### 阶段 5：同步与维护

- 建立 GitHub 远程仓库。
- 配置自动部署。
- 写清楚未来如何新增项目、更新图片和同步发布。

## 需要确认的问题

3. 第一版要展示几个项目：先展示 `2605-和平博弈`，后续逐步加入 lab 里的其他项目。
4. 要不要放简历下载：第一版不放。
5. 要不要放写作、论文、研究笔记或文章：第一版不放独立栏目。
6. 网站语言是中文、英文，还是中英双语：第一版中文。
7. 联系方式准备公开哪些：邮箱、GitHub、微信、LinkedIn、公众号或其他？
8. 风格更希望偏“实验室档案”“有趣作品集”“克制技术感”，还是混合？
9. 是否有不希望公开的内容边界？
10. 第一版上线时间有没有目标，比如一周内、一个月内，还是慢慢做？
