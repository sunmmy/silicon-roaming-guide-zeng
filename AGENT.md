# Agent 工作要求

本文件是该个人网站项目的核心协作说明。任何后续 Agent 进入项目时，应先阅读本文件，再查看 `docs/` 下的项目文档。

## 项目目标

建设一个简洁、长期可维护的个人展示网站，用来呈现个人背景、项目作品、研究/写作成果和联系方式。

## 工作原则

- 保持结构简单，避免过早引入复杂框架或过度工程化。
- 优先维护清晰内容，再做视觉和交互。
- 每次修改都应尽量留下可追踪的文档更新。
- 不随意删除已有内容；如果内容过时，优先移动到归档或说明原因。
- 网站设计应服务于个人展示，不做营销感过强的落地页。

## 必读文档

- `docs/background.md`：为什么做这个网站。
- `docs/progress.md`：目前已经完成了什么。
- `docs/next-steps.md`：下一步要做什么。
- `docs/tech-plan.md`：推荐技术方案。

## 内容维护规则

- 项目资料统一放在 `content/projects/`。
- 每个项目一个 Markdown 文件，使用 `content/projects/_template.md` 作为模板。
- 图片、截图、封面等素材放在 `assets/images/`。
- 不确定是否要公开的内容，先放在草稿里，不直接进入正式展示。
